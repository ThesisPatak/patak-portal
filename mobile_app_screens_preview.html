üéØ Key Updates & Features

Device Whitelist System: Users can only register from ESP32-001 to ESP32-005. Random device IDs are rejected with helpful error messages.
Improved Device Registration: Device selector UI shows available devices with location and availability status. Users can click to select or manually enter.
Removed Manage Devices: Dashboard no longer has "Manage Devices" button since device registration is a one-time setup step before accessing dashboard.
Device Ownership Prevention: If a device is already claimed by another user, it shows as "Claimed" in red and returns 409 Conflict error.
Real-time Device Status: Shows device locations (Main House, Secondary House, Office, etc.) and availability immediately.
Auto-populate Fields: Clicking a device in the selector automatically fills Device ID and House fields.
Security Validation: Readings endpoint validates device exists before accepting data. Prevents phantom readings from non-existent devices.
Better Error Handling: Device registration errors now include list of available devices to guide users.
Mobile-First Design: All screens optimized for mobile with touch-friendly buttons and responsive layout.
Real-time Updates: Dashboard uses SSE (Server-Sent Events) for real-time readings and status updates.
